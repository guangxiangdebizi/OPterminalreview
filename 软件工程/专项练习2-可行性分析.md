# 专项练习2：可行性分析

## 知识点总结

### 1. 可行性分析的四个方面

#### (1) 技术可行性
- 现有技术能否实现系统功能？
- 开发团队是否具备相应技术能力？
- 技术风险如何？

#### (2) 经济可行性
- 投资回报分析（成本-效益分析）
- 投资回收期
- 投资回报率（ROI）
- 净现值（NPV）

#### (3) 操作可行性（社会可行性）
- 用户是否愿意使用新系统？
- 系统是否符合组织文化？
- 用户培训是否可行？
- 是否会引起组织变革阻力？

#### (4) 法律可行性
- 是否符合法律法规？
- 是否侵犯知识产权？
- 数据隐私保护是否合规？

### 2. 经济可行性计算公式

#### 投资回收期（Payback Period）
```
投资回收期 = 初始投资 / 年均净收益
```

#### 投资回报率（ROI）
```
ROI = (总收益 - 总成本) / 总成本 × 100%
```

#### 净现值（NPV）
```
NPV = Σ [第t年净现金流 / (1+折现率)^t] - 初始投资
```

## 练习题目

### 题目背景

某连锁超市集团计划开发一套**智能库存管理系统（SIMS）**，以替代现有的半手工库存管理方式。现有方式存在以下问题：
- 库存盘点耗时长，误差率高（约5%）
- 缺货和积压问题频发，每年因此损失约80万元
- 采购决策依赖人工经验，缺乏数据支持

**新系统规划：**

**功能需求：**
1. 实时库存监控（RFID自动识别）
2. 智能补货提醒（基于销售预测）
3. 供应商管理
4. 多门店库存调配
5. 数据分析与报表

**成本估算：**
- 开发成本：120万元（软件开发80万，硬件设备40万）
- 年运维成本：15万元（服务器、网络、人员）
- 员工培训成本：8万元（一次性）

**收益估算：**
- 减少缺货和积压损失：每年节省60万元
- 提高库存周转率，降低资金占用成本：每年节省25万元
- 减少人工盘点成本：每年节省12万元
- 提高采购效率：每年节省8万元

**其他信息：**
- 系统预计使用寿命：5年
- 折现率：10%
- 开发团队有Java、Python经验，但无RFID技术经验
- 现有员工平均年龄45岁，对新技术接受度较低
- 系统需要收集顾客购买数据用于销售预测

### 问题

1. **经济可行性分析**（10分）
   - (1) 计算系统的投资回收期（3分）
   - (2) 计算5年期的投资回报率（ROI）（3分）
   - (3) 计算净现值（NPV），并判断项目是否经济可行（4分）

2. **技术可行性分析**（4分）
   - 指出该项目可能存在的技术风险（2分）
   - 提出降低技术风险的建议（2分）

3. **操作可行性分析**（3分）
   - 指出可能影响系统实施的操作可行性问题（2分）
   - 提出解决建议（1分）

4. **法律可行性分析**（3分）
   - 指出项目可能涉及的法律合规问题（2分）
   - 提出合规建议（1分）

## 标准答案

### 1. 经济可行性分析（10分）

#### (1) 投资回收期（3分）

**计算过程：**

初始投资 = 开发成本 + 培训成本 = 120 + 8 = **128万元**

年收益 = 60 + 25 + 12 + 8 = **105万元/年**

年成本 = 运维成本 = **15万元/年**

年净收益 = 105 - 15 = **90万元/年**

```
投资回收期 = 初始投资 / 年净收益
          = 128 / 90
          ≈ 1.42年
```

**答案：投资回收期约为 1.42年（或1年5个月）**（3分）

**评分标准：**
- 正确计算初始投资（1分）
- 正确计算年净收益（1分）
- 正确计算投资回收期（1分）

---

#### (2) 投资回报率（3分）

**计算过程：**

5年期总收益 = 105 × 5 = **525万元**

5年期总成本 = 初始投资 + 5年运维成本
           = 128 + (15 × 5)
           = 128 + 75
           = **203万元**

5年期净收益 = 525 - 203 = **322万元**

```
ROI = (总收益 - 总成本) / 总成本 × 100%
    = (525 - 203) / 203 × 100%
    = 322 / 203 × 100%
    ≈ 158.6%
```

**答案：5年期投资回报率约为 158.6%**（3分）

**评分标准：**
- 正确计算总成本（1分）
- 正确计算总收益（1分）
- 正确计算ROI（1分）

---

#### (3) 净现值（NPV）（4分）

**计算过程：**

折现率 r = 10% = 0.1

年净现金流 = 90万元（第1-5年）

```
NPV = Σ [年净现金流 / (1+r)^t] - 初始投资
    = 90/(1.1)^1 + 90/(1.1)^2 + 90/(1.1)^3 + 90/(1.1)^4 + 90/(1.1)^5 - 128
```

**逐年计算：**

| 年份 | 净现金流 | 折现系数 | 现值 |
|-----|---------|---------|------|
| 1 | 90 | 1/(1.1)^1 = 0.9091 | 81.82 |
| 2 | 90 | 1/(1.1)^2 = 0.8264 | 74.38 |
| 3 | 90 | 1/(1.1)^3 = 0.7513 | 67.62 |
| 4 | 90 | 1/(1.1)^4 = 0.6830 | 61.47 |
| 5 | 90 | 1/(1.1)^5 = 0.6209 | 55.88 |
| **合计** | | | **341.17** |

```
NPV = 341.17 - 128 = 213.17万元
```

**答案：净现值（NPV）= 213.17万元**（3分）

**结论：NPV > 0，项目经济可行**（1分）

**评分标准：**
- 正确使用NPV公式（1分）
- 正确计算各年现值（1分）
- 正确计算NPV总值（1分）
- 正确判断经济可行性（1分）

---

### 2. 技术可行性分析（4分）

#### (1) 技术风险（2分）

1. **RFID技术缺乏经验**（1分）
   - 开发团队无RFID技术经验，可能导致硬件集成困难、识别准确率不达标

2. **系统集成复杂度高**（1分）
   - 需要集成多门店数据、供应商系统、RFID设备，技术架构复杂

#### (2) 降低风险建议（2分）

1. **引入外部技术支持**（1分）
   - 聘请RFID领域专家或与成熟RFID供应商合作
   - 进行技术培训，提升团队能力

2. **采用分阶段实施策略**（1分）
   - 先在1-2家门店试点，验证技术方案
   - 积累经验后再全面推广

---

### 3. 操作可行性分析（3分）

#### (1) 操作可行性问题（2分）

1. **员工年龄偏大，技术接受度低**（1分）
   - 平均年龄45岁的员工可能抗拒使用新系统，影响系统推广

2. **工作流程变革阻力**（1分）
   - 从半手工到全自动，工作习惯改变大，可能引起抵触

#### (2) 解决建议（1分）

- **加强培训和激励**：分批次培训，设立"系统使用标兵"奖励（0.5分）
- **渐进式变革**：保留一定手工操作作为过渡，逐步提高自动化比例（0.5分）

---

### 4. 法律可行性分析（3分）

#### (1) 法律合规问题（2分）

1. **个人信息保护**（1分）
   - 系统收集顾客购买数据用于销售预测，涉及《个人信息保护法》

2. **数据安全责任**（1分）
   - 多门店数据传输和存储，需符合《数据安全法》和《网络安全法》

#### (2) 合规建议（1分）

- **数据匿名化处理**：销售预测仅使用脱敏后的统计数据，不存储顾客身份信息（0.5分）
- **完善安全措施**：采用数据加密传输、访问权限控制、定期安全审计（0.5分）

## 解题技巧

### 1. 经济可行性计算注意事项

| 易错点 | 正确做法 |
|-------|---------|
| 混淆一次性成本和年度成本 | 培训成本是一次性的，运维成本是每年的 |
| 忘记扣除年度成本 | 年净收益 = 年收益 - 年成本 |
| NPV计算不折现 | 必须用折现系数计算现值 |
| ROI计算期混淆 | 明确是计算几年期的ROI |

### 2. 可行性分析答题结构

```
问题识别 → 原因分析 → 解决方案 → 结论
```

**示例：**
- **问题**：员工年龄偏大，技术接受度低
- **原因**：长期使用传统方法，学习新系统有困难
- **方案**：分批培训 + 激励机制 + 渐进式变革
- **结论**：通过妥善措施可以解决，操作可行

### 3. 四个可行性的关注重点

| 可行性类型 | 关注重点 | 典型问题 |
|----------|---------|---------|
| 技术可行性 | 技术难度、团队能力 | 新技术、技术整合 |
| 经济可行性 | 成本、收益、回收期 | ROI、NPV |
| 操作可行性 | 用户接受度、组织适应性 | 变革阻力、培训需求 |
| 法律可行性 | 法律法规、合规性 | 隐私保护、知识产权 |

### 4. NPV快速计算技巧

对于等额现金流，可使用年金现值系数：

```
PVA = 年金 × [(1 - (1+r)^-n) / r]
```

本题中：
```
PVA = 90 × [(1 - 1.1^-5) / 0.1]
    = 90 × 3.7908
    = 341.17万元
NPV = 341.17 - 128 = 213.17万元
```

（提示：年金现值系数表在考试中可能会提供，也可能需要手算）
